# Organizacion local del proyecto

Ubicacion maestra:

`D:\GGLabs\01_PROJECTS\AI-Learning-Vault-OS`

## Carpetas

- `01_PROYECTO_LOCAL`: copia limpia y ejecutable del codigo.
- `02_DOCUMENTOS/01_RECIBIDOS`: archivos originales recibidos sin modificar.
- `02_DOCUMENTOS/02_WORD_EDITABLES`: documentos `.docx` de trabajo.
- `02_DOCUMENTOS/03_PDF_ESTUDIO`: PDFs finales o de consulta.
- `02_DOCUMENTOS/04_MARKDOWN_FUENTE`: fuentes editables para documentos.
- `02_DOCUMENTOS/05_REFERENCIAS`: material complementario.
- `03_ASSETS`: imagenes, diagramas y portadas.
- `04_ENTREGABLES`: borradores, aprobados y exportaciones finales.
- `05_ARCHIVO/POR_FECHA`: versiones historicas cerradas.
- `06_BACKUPS_MANUALES`: respaldos puntuales antes de cambios grandes.

## Convencion de nombres

Usar:

`AAAA-MM-DD_tipo_titulo_v01.ext`

Ejemplo:

`2026-06-05_pdf_guia-fundamentos-ia_v01.pdf`

## Reglas

- Mantener originales intactos en `01_RECIBIDOS`.
- Guardar editables Word en `02_WORD_EDITABLES`.
- Guardar PDFs aprobados en `04_ENTREGABLES/02_APROBADOS`.
- No guardar secretos, API keys, datos privados ni videos pesados dentro del proyecto.
